Архивирование данных

Архивирование - операция, в результате которой программа Erags упаковывает (сжимает) файлы базы данных и помещает их в файл с именем eradb.zip. Если включён режим автоматического архивирования, то программа Erags автоматически предлагает архивирование данных в указанный день недели или каждый день. Если режим автоматического архивирование выключен, то файлы базы данных можно заархивировать, выбрав пункт меню "Архівувати БД ЕРАГС у zip-архів", как показано на рисунке:



Архивирование повышает степень сохранности данных. При аварийном отключении электроэнергии структура файла базы данных может быть разрушена и может случиться так, что программа Erags не сможет открыть файл базы данных и, таким образом, данные будут безвозвратно потеряны. Если же файлы базы данных периодически архивируется, то есть возможность восстановить файл базы данных из архивного файла. Для этого нужно выбрать пункт "Відновити файл(и) БД ЕРАГС із zip-архіву" из меню "Файл":



Появится диалоговое окно со списком заархивированных файлов базы данных. Выберите из этого списка файл, который не открывается и нажмите кнопку "Так".


Операция восстановления файла базы данных из архива заменит соответствующий файл базы данных на жёстком диске его архивной копией. Поэтому восстанавливать файл базы данных из архива нужно только в том случае, если он не открывается программой Erags.

Архивный файл eradb.zip рекомендуется время от времени записывать на какой-либо носитель информации: компакт-диск, флэш-носитель и т.п. В этом случае база данных будет сохранена на независимом носителе, что позволяет восстановить данные даже в случае, если жёсткий диск Вашего компьютера пришёл в негодность.

Архивный файл eradb.zip можно передавать в областной отдел для пополнения базы данных областного отдела. В областном отделе данные можно распаковать и использовать их для поиска или для выдачи повторных свидетельств.
