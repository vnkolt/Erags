Архівування даних

Архівування - операція, в результаті якої програма Erags пакує (стискує) файлы бази даних і поміщає їх у архівний файл eradb.zip. Якщо режим автоматичного архівування увімкнуто, програма Erags автоматично пропонуматиме архівування даних у визначений день тижня або щодня. Якщо режим автоматичного архівування виключений, то файли бази даних можна заархівувати, обравши пункт меню "Архівувати БД ЕРАГС у zip-архів", як показано малюнку:



Архівування підвищує ступінь збереження даних. У разі аварійного відключення електроенергії структура файла бази даних може бути зруйнована і може статися, що програма Erags не зможе відкрити файл бази даних і, таким чином, дані будуть безповоротно втрачені. Якщо ж файли бази даних періодично архівуються, є можливість відновити файл бази даних із архівного файлу. Для цього треба вибрати пункт "Відновити файл(и) БД ЕРАГС з zip-архіву" з меню "Файл":



З'явиться діалогове вікно зі списком заархівованних файлів бази даних. Виберіть зі списку файл, який не відкривається і натисніть кнопку "Так".


Операція відновлення файла бази даних із архіву замінить відповідний файл бази даних на жорсткого диску його архівною копією. Тому відновлювати файл бази даних із архіву потрібно у разі, якщо він пошкоджений і не відкривається програмою Erags.

Архівний файл eradb.zip рекомендується раз у раз записувати на будь-який носій інформації: компакт-диск, флеш-носій тощо. У цьому разі база даних буде збережено на незалежному носії, що дозволяє відновити дані навіть тоді, якщо жорсткий диск Вашого комп'ютера прийшов у непридатність.

Архівний файл eradb.zip можна передавати до обласного відділу для поповнення бази даних обласного відділу. У обласному відділі дані можна розпакувати і використовувати задля пошуку або видачі повторних свідоцтв.
