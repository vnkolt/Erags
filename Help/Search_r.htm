Поиск

Программа Erags позволяет осуществлять поиск данных двумя способами:

1. Поиск внутри текущего файла БД по текущему виду регистрации.

2. Поиск в файлах БД по указанному маршруту и заданным видам регистрации.

Для поиска внутри текущего файла БД по текущему виду регистрации нажмите комбинацию клавиш Ctrl+G или кнопку .

Появится окно с результатами выборки, в которую войдут записи из текущего файла БД и текущего вида регистрации. На картинке ниже показано это окно с примером выборки по рождениям за 2007-й год:

Результаты выборки можно сортировать по столбцам "№", "Дата", "Прізвище". Для этого наведите курсор на заголовок соответствующего столбца и щелкните левой кнопкой мыши. Кроме того можно использовать текстовые поля в верхней части окна для поиска по номеру или по фамилии. В примере как раз показан ввод в текстовом поле "ПІБ" строки "га" и соответстсвующая запись с фамилией "Галчик".

Поиск внутри текущего файла БД по текущему виду регистрации удобен лишь в случае, если Вы точно знаете, что искомая запись находится в определённом файле БД. В противном случае нужно воспользоваться поиском по указанному маршруту и заданным видам регистрации. В этом случае Erags будет сканировать файлы БД ЭРАГС в по указанному маршруту и искать в них записи, которые соответствуют заданиям поиска.

  • Маршрут поиска - папка (директория, каталог) в которой находятся файлы БД ЭРАГС.
  • Задание поиска - совокупность одной или более строк (частей условия), каждая из которых означает фамилию, имя и отчество.

    • Для поиска в файлах БД по указанному маршруту и заданным видам регистрации нажмите комбинацию клавиш Ctrl+F или кнопку .

    Появится окно с пустым списком заданий (слева) и пустым списком результатов поиска (справа):

    Укажите одно или более заданий для поиска. Каждое задание может состоять из трёх частей: фамилии, имени и отчества. Любая часть может быть пустой. На рисунке ниже показано окно для поиска с четырьмя заданиями:

    Обратите внимание, что на панели инструментов "Поиск" кнопка включена, т.е. установлен режим частичного совпадения данных.

  • Режим частичного совпадения данных - режим, при котором будут находиться такие записи, которые частично удовлетворяют всем условиям поиска. Например, в режиме частичного совпадения для задания с условиями "Фамилия = Ткач, Имя = Олекс" будут находится записи, в которых есть фамилия с подстрокой Ткач и имя с подстрокой Олекс: Ткачов Олександр Іванович, Ткачук Олексій Петрович, Ткаченко Олександра Андріївна и т.д.

  • Режим полного совпадения данных - режим, при котором будут находиться только такие записи, которые полностью удовлетворяют всем условиям поиска. Например, в режиме полного совпадения для задания с условиями "Фамилия = Ткач, Имя = Олександр" будут находится записи, в которых есть фамилия Ткач и имя Олександр: Ткач Олександр Іванович, Ткач Олександр Петрович и т.д.

    • После того, как задания для поиска определены, нажмите на панели инструментов "Поиск" кнопку "Начать поиск" , , чтобы начать поиск записей.

    Поиск может продолжаться от нескольких секунд до нескольких минут, в зависимости от количества заданий, количества файлов БД, количества записей в файлах БД, режима поиска, быстродействия компьютера и т.п. После завершения поиска в правой части окна будет показан список результатов, т.е. найденные записи с указанием вида регистрации (столбец "АЗ"), номера актовой записи (столбец "№"), ФИО (столбец "Прізвище - ім'я - по батькові") и полного имени файла БД, в котором находится найденная запись (столбец "Файл"). По каждому столбцу, за исключением "АЗ" можно произвести сортировку. Для этого надо кликнуть на соответствующем столбце.
