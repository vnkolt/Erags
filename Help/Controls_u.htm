Елементи введення

Для введення даних у ПК ЕРАГС використовуються наступні елементи введення:

Елемент Зовнішній вигляд Коментар
Текстове поле Текстове поле використовується для введення текстових даних. Поля номера актового запису, прізвища, приміток, номера будинку, квартири - це все текстові поля. Деякі текстові поля автоматично змінюють текст під час введення. Наприклад, при введенні серії паспорта букви автоматично перетворяться у верхній регістр
Поле дати

Поле дати використовується, коли потрібно визначити дату. Дата реєстрації, дата народження, дата смерті, дата видачі паспорта є полями дати. Коли поле дати є поточним елементом введення (білий фон) показані тільки цифри і крапки, наприклад 29.07.2004. Якщо поточний елемент введення інший, поле дати відображається інакше, наприклад 29 липня 2004. Дату потрібно вводити тільки цифрами. Крапки чи інші роздільники вводити не потрібно. Якщо рік дати, що вводиться, більше, ніж 1929 (установка операційної систему за умовчаванням), то можна вводити дві цифри року. У цьому випадку значення року 53, 64, 87 будуть інтерпретуватися відповідно як 1953, 1964 і 1987, а значення 03 буде інтерпретуватися як 2003. Для введення значення року до 1930 потрібно вводити чотири цифри року.
Поле зі списком

Поле зі списком використовується у випадках, коли можливо обрати якесь значення з деякої сукупності можливих значень. Використання поля зі списком дозволяє значно скоротити час на введення даних. Для деяких даних при вході в поле зі списком ЕРАГС автоматично пропонує деяке значення, яке називається значенням за умовчаванням. Будь-яке поле зі списком можна розкрити за допомогою миші або комбінації клавіш Alt+в. Однак, значення поля зі списком підбирається і при звичайному клавіатурному введенні. Наприклад, якщо потрібно ввести по батькові "Іванівна" і таке значення є в списку, досить ввести одну чи дві перші букви. Дуже ефективно поле зі списком можна використовувати для визначення установ, що видають медичні свідоцтва про смерть чи про народження, а також для визначення причини смерті. У цих випадках можна використовувати псевдоніми, що дозволяє визначити досить довгу назву за допомогою натискання двох-трьох клавіш.
Перемикач

Перемикач використовується там, де потрібно обрати значення з двох або більш можливих, кількість яких обмежена і не може бути змінена. Типові приклади перемикачів - поле для визначення статі новонародженого чи померлого, тип паспортного документа, підстава запису відомостей про батька. У деяких випадках зміна значення перемикача приводить до зміни даних інших елементів введення.
Керуюча кнопка

Керуюча кнопка використовуються в різних діалогових вікнах і панелях інструментів. Керуюча кнопка не змінює даних, вона лише дозволяє викликати діалогове вікно або меню, підтвердити введення, тощо.
Прапорець

Прапорець використовується для включення/вимикання деякої властивості чи ознаки в деяких діалогових вікнах. Приклад - ознака повторного свідоцтва в діалоговому вікні "Установи друку свідоцтва".

Група перемикачів

Група перемикачів використовується в деяких діалогових вікнах для вибору деякого значення із сукупності припустимих значень. Приклад - діалогове вікно "Шаблони Документів".
Список

Список використовується для вибору деякого значення з досить великої сукупності можливих значень. Списки можуть бути з одним чи декількома стовпцями. Стовпці списків можуть бути озаглавлені. Приклад - список вибору шаблона документа в діалоговому вікні "Шаблони Документів".
Деревоподібний список

Деревоподібний список використовується для вибору деякого значення зі списку, що має деревоподібну структуру. Деревоподібний список має тільки один стовпець. Елемент деревоподібного списку може мати невелике зображення для кращого асоціативного сприйняття. Приклади - форми "Список географічних назв", "Список органів реєстрації", діалогове вікно "Установи друку свідоцтва".
