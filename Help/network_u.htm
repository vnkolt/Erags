Робота з ПК ЕРАГС в локальній комп'ютерній мережі.

Якщо у відділі РАГС є кілька комп'ютерів, то для ефективної роботи з ПК ЕРАГС їх необхідно об'єднати в загальну локальну комп'ютерну мережу. Як основний комп'ютер (тобто такий, на якому будуть розташовані файли БД ЕРАГС) варто обрати найбільш потужний комп'ютер з наявних у відділі.

Під час установки ПК ЕРАГС на клієнтський (не основний) комп'ютер слід вказати "Другорядний", як показано на малюнку:



Щоб відкрити файл БД ПК ЕРАГС на клієнтському комп'ютері у файловому діалозі слід обрати Сетевое Окружение, потім вказати ім'я групи (на малюнку група називається Rags), потім відкрити основний комп'ютер (на малюнку MainPC), потім розкрити папку з файлами бази даних і вибрати потрібний файл.



Варто пам'ятати, що на основному комп'ютері повинний бути відкрит доступ по читанню і запису до папки, де розташовані файли БД ПК ЕРАГС.
Для нормальної експлуатації ПК ЕРАГС основний комп'ютер варто включати першим, а виключати останнім.

Основна програма ПК ЕРАГС (Erags) у заголовку головного вікна завжди пише повне ім'я поточного файлу бази даних. Якщо файл БД відкритий з іншого комп'ютера, то імені файли передує ім'я комп'ютера, де цей файл знаходиться або ім'я мережового логічного диску:



Програма Erags після першого старту завжди намагається відкрити останній файл БД, з яким вона працювала. Файл не буде відкритий, якщо порушене мережне з'єднання, якщо було змінено ім'я файлу, файл було знищено, якщо було змінено ім'я основного комп'ютеру або основний комп'ютер не був включений, тощо.
